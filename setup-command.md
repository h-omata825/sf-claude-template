Salesforce + Claude Code の新規プロジェクトをセットアップします。
以下の手順を順番に実行してください。

---

## 事前チェック

まず以下のコマンドを実行して、必要なツールがインストールされているか確認する:

```bash
sf --version
git --version
```

- `sf` が見つからない場合 → 「Salesforce CLIがインストールされていません。 https://developer.salesforce.com/tools/salesforcecli からインストールしてください」と案内して中断
- `git` が見つからない場合 → 「Gitがインストールされていません」と案内して中断

---

## Step 1: ヒアリング

ユーザーに以下を **1つずつ** 質問する。前の回答を得てから次を聞く。

### 1-1. プロジェクト名
```
プロジェクト名を教えてください（英語、ハイフン・キャメルケース等）
例: myProject, sales-portal, CaseManagement
```

### 1-2. 作成先フォルダ
```
プロジェクトフォルダの作成先パスを教えてください
例: C:/workspace, D:/projects
```
- 入力されたパスが存在するか確認する。存在しなければ「そのフォルダは存在しません。作成しますか？」と聞く

### 1-3. テンプレートURL
```
テンプレートのGitリポジトリURLを教えてください
デフォルト: https://github.com/h-omata825/sf-claude-template.git
（デフォルトでよければそのままEnterしてください）
```
- 入力がなければデフォルト値を使う

### 1-4. 組織認証
```
Salesforce組織に接続しますか？
  1. 本番/Developer Edition（login.salesforce.com）
  2. Sandbox（test.salesforce.com）
  3. スキップ（後で設定する）
```
- 1 または 2 を選んだ場合 → 「組織のエイリアスを教えてください（例: myOrg）」と追加で聞く

---

## Step 2: 確認

ヒアリング内容をまとめて表示し、確認を取る:

```
以下の内容でセットアップします。よろしいですか？

  プロジェクト名: <プロジェクト名>
  作成先: <作成先>/<プロジェクト名>
  テンプレート: <URL>
  組織認証: <エイリアス> or スキップ

（はい / いいえ）
```

「いいえ」の場合はStep 1に戻る。

---

## Step 3: プロジェクト作成

以下のコマンドを順番に実行する。各ステップの完了を確認してから次に進む。

### 3-1. テンプレート取得
```bash
git clone <テンプレートURL> "<作成先>/_sf-template-tmp"
```

### 3-2. セットアップ実行
```bash
bash "<作成先>/_sf-template-tmp/setup.sh" <プロジェクト名> "<作成先>"
```

### 3-3. 一時フォルダ削除
```bash
rm -rf "<作成先>/_sf-template-tmp"
```

各コマンドでエラーが出た場合:
- clone失敗 → 「URLを確認してください。プライベートリポジトリの場合はGitの認証設定が必要です」
- setup.sh失敗 → エラー内容を表示して原因を説明
- プロジェクト名が既存 → 「既に存在します。別の名前にしますか？」と聞く

---

## Step 4: 組織認証（スキップでなければ）

```bash
cd "<作成先>/<プロジェクト名>"
```

選択に応じて実行:
- 本番の場合: `sf org login web -a <エイリアス>`
- Sandboxの場合: `sf org login web -a <エイリアス> -r https://test.salesforce.com`

ユーザーに伝える:
```
ブラウザが開きます。Salesforceにログインしてください。
ログインが完了したらこちらに戻ってきてください。
```

認証完了後に確認:
```bash
sf org display -o <エイリアス>
```

成功したら「組織の認証が完了しました」と伝える。
失敗したら「認証に問題がありました。もう一度試しますか？」と聞く。

---

## Step 5: 完了報告

```
============================================
  セットアップ完了！
============================================

プロジェクト: <作成先>/<プロジェクト名>
組織: <エイリアス>（認証済み / 未設定）

次のステップ:
  1. VSCodeでプロジェクトフォルダを開く
     code "<作成先>/<プロジェクト名>"

  2. CLAUDE.md を開いてプロジェクト固有情報を記入
     - プロジェクト概要
     - 主要オブジェクト
     - 命名規則
     - 注意事項

  3. .mcp.json のトークンを設定（必要な場合）
     - GitHub: Personal Access Token
     - Slack: Bot Token
     - Notion: Integration Token
```
