## 🔴 Critical（2件）

### [C-1] sf-design-writer.md：`--feature-groups` が `generate_feature_list.py` の argparse に未定義

**問題**: `sf-design-writer.md:447` および `:456` の Bash ブロックで `--feature-groups "{project_dir}/docs/.sf/feature_groups.yml"` を `generate_feature_list.py` に渡しているが、スクリプトの argparse（lines 330–344）に `--feature-groups` は定義されていない。
**原因**: feature_groups.yml は `check_feature_groups.py` 等他スクリプト向けの設計情報ファイルで、`generate_feature_list.py` には関係しない。agent 実装時に誤って付加したと思われる。
**影響**: 実行時に `error: unrecognized arguments: --feature-groups` が出て xlsx 生成が完全に失敗する。
**修正案**: `sf-design-writer.md:447` と `:456` の `--feature-groups ...` 行を削除する（スクリプトへの機能追加ではなく、不要な引数を agent 側から除去するのが最小変更）。
**補足**: `sf-design-step3.md:67` の `generate_feature_list.py` 呼び出し（前回 C-3 修正で書き換え済み）には `--feature-groups` は含まれておらず問題なし。修正対象は sf-design-writer.md の2箇所のみ。

### [C-2] sf-basic-design-writer.md / sf-domain-design-writer.md：参照テンプレート xlsx が物理的に存在しない

**問題**: `sf-basic-design-writer.md:333` が `scripts/python/sf-doc-mcp/基本設計書テンプレート.xlsx` を `--template` 引数で参照し、`sf-domain-design-writer.md:335` が `ドメイン設計書テンプレート.xlsx` を参照しているが、両ファイルとも `scripts/python/sf-doc-mcp/` に存在しない。
**原因**: テンプレート生成スクリプト（`build_basic_design_template.py` / `build_domain_design_template.py`）が用意されているが、一度も実行されていないため xlsx が未生成のままリポジトリに含まれていない。
**影響**: `generate_basic_design.py` / `generate_domain_design.py` が `FileNotFoundError: テンプレートが見つかりません` で落ちる。基本設計書・ドメイン設計書の生成が完全に機能しない。
**修正案**: build スクリプトをそれぞれ実行して xlsx を生成し、リポジトリにコミットする。`build_basic_design_template.py --output .../基本設計書テンプレート.xlsx` / `build_domain_design_template.py --output .../ドメイン設計書テンプレート.xlsx`。
**補足**: 既存の他テンプレート（機能一覧テンプレート.xlsx、詳細設計書テンプレート.xlsx等）は同梱済みで問題なし。

## 🟡 Warning（1件）

### [W-1] generate.py / list_objects.py：`shell=True` + f-string でシェルインジェクション懸念

**問題**: `generate.py:22–25` と `list_objects.py:15–18` で `subprocess.run(f"sf org display --target-org {alias} --json", ..., shell=True)` を使用。alias の値にシェルメタ文字（`;`, `&&`, `|` 等）が含まれるとコマンドインジェクションが成立する。
**原因**: subprocess でシェル文字列結合が慣用的に使われたが、list 形式との使い分けが徹底されていない。
**影響**: 悪意ある alias 値（例: `alias; rm -rf /`）を渡された場合に任意コマンドが実行される。テンプレートを配布リポジトリとして使うため、外部ユーザーが alias を自由に設定できる。
**修正案**: `shell=False` に変更し、コマンドを list 形式に書き換える。`subprocess.run(["sf", "org", "display", "--target-org", alias, "--json"], ...)` で同等の動作が得られる。`shutil.which("sf")` で PATH 解決も確認すること。
**補足**: `list_objects.py` は現時点で agent からは呼ばれていない（dead）が、リポジトリに含まれるため修正対象とする。`generate.py` は `sf-doc-objects-writer.md` から実際に呼ばれる。
