# /backlog-report [課題ID]

Backlog 課題の対応工数レポートを生成する。課題IDを指定すると1件の詳細レポート、省略すると全件サマリーを出力する。

## 引数

- `[課題ID]`（省略可）— 指定した場合は該当課題の詳細レポート。省略した場合は `docs/effort-log.md` の全件サマリー

---

## Step 1. データ取得

### 課題IDを指定した場合

1. `docs/effort-log.md` から該当行を読む
2. Backlog MCP で課題情報を取得する（件名・対応日・担当者・コメント）
3. 見込み・実績の両方が記録されているか確認する（実績未記録の場合はその旨を明示）

### 課題IDを省略した場合

`docs/effort-log.md` を全件読み込む。

---

## Step 2. レポート生成

### 1件詳細レポート（課題ID指定時）

```
## 工数レポート — [課題ID] [件名]

### 基本情報
| 項目 | 内容 |
|---|---|
| 課題ID | [ID] |
| 件名 | [件名] |
| 対応種別 | バグ / 追加要望 / その他 |
| 対応日 | YYYY-MM-DD |

### 工数比較
| 区分 | 工数 |
|---|---|
| 見込み（ClaudeCode 使用） | Xh |
| 見込み（ClaudeCode 未使用） | Xh |
| 実績（ClaudeCode 使用） | Xh |
| 見込み精度 | ±X%（見込みXh → 実績Xh） |

### 効率化効果
- ClaudeCode による削減時間: Xh（削減率 X%）
- 見込みの精度: [高 / 中 / 低]（誤差 X%）

### コメント
[見込みと実績の乖離がある場合はその理由を考察]
```

### 全件サマリーレポート（課題ID省略時）

```
## 工数サマリーレポート

生成日時: YYYY-MM-DD

### 全課題一覧
| 課題ID | 件名 | 種別 | 見込み（CC） | 見込み（非CC） | 実績（CC） | 削減効果 | 対応日 |
|---|---|---|---|---|---|---|---|
| [ID] | [件名] | バグ | Xh | Xh | Xh | X% | YYYY-MM-DD |
| ...（実績未記録の場合は「-」） |

### 集計（実績記録済み課題のみ）
| 指標 | 値 |
|---|---|
| 対応課題数（実績あり） | X件 |
| CC使用 合計工数 | Xh |
| CC未使用 見込み合計 | Xh |
| 削減時間合計 | Xh |
| 平均削減率 | X% |
| 見込み精度（平均誤差） | ±X% |

### 種別内訳
| 種別 | 件数 | CC使用平均工数 | CC未使用平均工数 | 平均削減率 |
|---|---|---|---|---|
| バグ | X件 | Xh | Xh | X% |
| 追加要望 | X件 | Xh | Xh | X% |
| その他 | X件 | Xh | Xh | X% |

### 所見
[傾向・特記事項。例:「バグ対応はCC使用で平均4倍の効率化。追加要望は調査コストが高く2倍程度」]
```

---

## Step 3. 出力

レポートをターミナルに出力する。

必要に応じてファイル保存する場合は `docs/reports/effort-report-YYYYMMDD.md` に保存する。

---

## 使用例

```
/backlog-report             # 全件サマリー
/backlog-report GF-327      # GF-327の詳細レポート
```
