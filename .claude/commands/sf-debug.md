maintenanceエージェントとして、以下のバグ・エラーを調査・修正してください。

## エラー内容・症状
$ARGUMENTS

## 調査手順
1. エラーメッセージ・スタックトレースを確認する
2. 影響範囲（ユーザー・機能・環境）を特定する
3. 再現手順を確認する
4. デバッグログが必要な場合は取得方法を案内する
5. 根本原因を特定する
6. 暫定対応（緊急回避策）を提案する
7. 恒久対応（根本修正）を提案する

## 出力形式
影響 → 原因 → 暫定対応 → 恒久対応 → 再発防止 の順で報告してください。

## 注意事項
- 本番環境への修正デプロイは必ずユーザー確認を取ってから実行する
- エラーログや個人情報を出力に含めない
