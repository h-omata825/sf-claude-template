assistantエージェントとして、テンプレートのアップグレードを実行してください。

## 概要

`scripts/upgrade.sh` を実行してテンプレートを更新する。
スクリプトが差分検出・確認・適用・クリーンアップを全て行う。

## ユーザー入力

$ARGUMENTS

- **引数なし** → デフォルト（mainブランチ）
- **タグ/ブランチ指定** → そのバージョンを取得

## 実行

```bash
bash scripts/upgrade.sh -y $ARGUMENTS
```

`-y` フラグで「適用しますか？」の確認を自動承認する。
テンプレートから削除されたファイルは **自動削除される**（対象: `.claude/agents/` / `.claude/commands/` / `.claude/templates/` / `scripts/` 配下のファイル。`.claude/CLAUDE.md` / `.claude/settings.json` / `.gitignore` は上書きコピーのみで削除検出対象外）。

> **なぜ `.claude/` に書き込めるか**:
> `settings.json` の deny ルールは `Bash` ツールに対する特定コマンドパターン（sf 系の本番組織保護）のみで、`.claude/` 配下の書き換え自体には制限がない。
> `bash scripts/upgrade.sh` 内の `cp` も `Bash(*)` の allow に含まれ、スクリプトが `.claude/` 配下を自由に上書きできる。
> テンプレート管理者がアップグレードを意図的に行う場合にのみ使うコマンドのため、この設計は意図的なもの。

## 実行後

スクリプトの出力結果をユーザーに伝え、以下のように対応する:

**変更があった場合**（"コミット完了" が出力に含まれる）:

```bash
git push origin HEAD
```

push 完了後:
```
✅ テンプレート更新 + push 完了
```

**変更がなかった場合**（"テンプレートは最新です" が出力に含まれる）:
```
✅ テンプレートは既に最新です。push は不要です。
```

**変更はあったが commit 対象外の場合**（"Git: コミット対象の変更なし" が出力に含まれる）:
```
✅ ファイルは更新済みですが Git 管理外 or 既に適用済みのため commit/push は不要です。
```

**スクリプトがエラーで終了した場合**（"[ERROR]" が出力に含まれる、または exit code が非 0）:
```
❌ テンプレート更新に失敗しました
```
スクリプトの `[ERROR]` メッセージをそのままユーザーに伝え、想定される原因（git 未インストール / プロジェクトルート外で実行（`.claude/` が存在しない） / URL・ブランチ/タグ指定の誤り / ネットワーク不通）を案内して、再試行または手動対応を促す。push は実行しない。

**上記いずれの文字列も出力に含まれない場合**:
スクリプトの出力をそのままユーザーに伝え、状況の確認と手動対応を提案する。判断がつかないまま push しない。

> **注意**: `.gitignore` に `.claude/` が含まれている場合、`git diff` に差分が表示されない。
> このテンプレートでは `.claude/` を Git 管理対象にしているため通常は問題ないが、
> もし `.gitignore` を手動編集して除外した場合はファイルを直接確認すること。
