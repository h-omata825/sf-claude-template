# 資料保存コマンド

添付・貼り付けされた資料をMarkdown形式に変換してdocsフォルダに保存する。

## 実行手順

以下の手順で処理すること。

### Step 1: カテゴリ判定

資料の内容からカテゴリを判定する。判定できない場合はユーザーに確認する。

| カテゴリ | 保存先 | 対象資料 |
|---|---|---|
| requirements | `docs/requirements/` | 要件定義書・ユーザーストーリー・ヒアリングメモ・RFP |
| design | `docs/design/` | 設計書・オブジェクト定義書・機能設計書・ER図・画面設計 |
| test | `docs/test/` | テスト計画・テストケース・バグ報告・品質レポート |
| minutes | `docs/minutes/` | 議事録・決定事項・アクションアイテム・MTGメモ |
| manuals | `docs/manuals/` | 手順書・マニュアル・操作ガイド・リリース手順 |

### Step 2: ファイル名生成

形式: `YYYYMMDD_<資料タイトルを英語スネークケースで>.md`

例:
- `20260402_order_requirements.md`
- `20260402_account_object_definition.md`
- `20260402_sprint3_minutes.md`

### Step 3: Markdown変換ルール

- 表はMarkdownテーブル形式に変換する
- 画像・図は`[図: キャプション]`と記載してスキップする
- セクション見出しは`##`から始める
- 元資料のファイル名・作成日・提供者が判明している場合はfrontmatterに記載する

```markdown
---
source: 元ファイル名（例: 要件定義書_v1.2.xlsx）
date: YYYY-MM-DD
author: 提供者名（不明な場合は省略）
saved: YYYY-MM-DD
---
```

### Step 4: 保存と確認

1. 変換後のMarkdownをファイルに書き出す
2. 保存パスをユーザーに報告する
3. `CLAUDE.md`の「プロジェクト資材の参照」セクションに記載されたパスと一致していることを確認する

## 使用例

```
/save-doc
（資料を貼り付けまたは添付）
```

または

```
この要件定義書を保存して
/save-doc
```

## 注意事項

- 機密情報（パスワード・トークン・個人情報）が含まれる場合は保存前にユーザーに確認する
- 既存ファイルと同名の場合は上書き前にユーザーに確認する
- 変換後に内容の重要な欠落がないかユーザーに確認を促す
