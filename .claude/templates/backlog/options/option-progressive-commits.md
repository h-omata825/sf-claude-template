# option-progressive-commits

## 何をするか

変更を細かい単位で段階コミットする。ロールバック容易性とレビュー性を高める。

## 実行手順

1. 実装計画（implementation-plan.md）の変更内容を論理的な単位に分割する:
   - 「オブジェクト定義（フィールド追加）」
   - 「Apex クラスの実装」
   - 「LWC コンポーネントの実装」
   - 「テストクラスの実装」
   - 「docs/ 更新」
2. 各単位ごとにコミットする:
   ```bash
   git add {対象ファイル}
   git commit -m "feat({issueID}): {変更の内容を簡潔に}"
   ```
3. コミット粒度の目安:
   - 1 コミット = 1 ファイルまたは密接に関連する 2〜3 ファイル
   - 「これを単独で revert したい」と思える粒度
4. コミットメッセージはその変更が「何をなぜしたか」を含める:
   - `feat({issueID}): {機能名} に {処理名} を追加`
   - `fix({issueID}): {症状} の原因となる {箇所} を修正`

## 出力

コミット後に実施したコミット一覧を implementation-plan.md に追記:

## 段階コミット一覧

| # | コミットハッシュ | 内容 |
|---|---|---|
| 1 | {hash} | {変更内容} |
| 2 | ... | ... |
