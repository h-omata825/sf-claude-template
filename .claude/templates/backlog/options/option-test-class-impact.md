# option-test-class-impact

## 何をするか

変更対象に関連する既存テストクラスを特定し、変更後にテストが壊れる可能性を事前評価する。

## 実行手順

1. 変更対象のクラス名・メソッド名を確定する
2. 関連テストクラスを Grep で特定する:
   ```bash
   Grep pattern: {クラス名 or メソッド名}
   ファイル: force-app/**/*Test*.cls, force-app/**/*_Test*.cls
   ```
3. 特定したテストクラスを Read して以下を確認する:
   - テスト対象メソッド（どの処理をテストしているか）
   - テストデータの作り方（変更後も同じデータで動くか）
   - assert の内容（変更後も同じ結果が期待できるか）
4. 壊れるリスクを評価する:
   - 安全（テストは壊れない）→ その根拠を記録
   - 修正が必要（テストの期待値・データを変える必要がある）→ 修正箇所を特定
5. 実際に `sf apex run test` で現時点のテスト状態を確認する（ベースライン）
