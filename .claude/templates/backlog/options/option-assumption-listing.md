# option-assumption-listing

## 何をするか

調査・修正方針に含まれる「暗黙の前提」を全て明示し、検証可能な形に変換する。「〇〇のはず」という表現を排除する。

## 実行手順

1. 調査レポート・修正方針案を読み返し、以下のパターンを探す:
   - 「〇〇のはず」「〇〇と思われる」「おそらく〇〇」
   - 「〇〇の場合は」（「〇〇の場合」が実際に成立しているか確認していない）
   - 「既存の仕組みを使えば〇〇」（その仕組みが今も同じように動くと仮定している）
2. 見つかった前提を全て列挙する（最低 3 個、多い場合は全件）
3. 各前提について確認手順を明示する:
   - コードで確認できる → Grep / Read で確認して結果を記録
   - Sandbox で確認できる → 実際に確認して結果を記録
   - 確認不可能 → 業務要件の不確実点に追加
4. 確認した結果を investigation.md に記録する

## 出力

investigation.md に追記:
