# option-staged-deployment-plan

## 何をするか

段階的デプロイ計画を策定する。全社影響・データ移行を含む変更を、リスクを分散して展開する。

## 実行手順

1. 変更の影響範囲を確認する（影響ユーザー数・オブジェクト・機能数）
2. 以下の段階展開パターンを検討する:

   **Phase 別**:
   - Phase 1: 開発・Sandbox 検証
   - Phase 2: ステージング（UAT 環境）での確認
   - Phase 3: 本番の一部ユーザーへの先行展開
   - Phase 4: 全ユーザーへの展開

   **ユーザー別（プロファイル・権限セット単位）**:
   - 管理者グループ → 担当グループ → 全社の順で展開
   - フィーチャーフラグで制御する場合は option-feature-flag-design と連携

   **機能別（機能フラグ）**:
   - 新機能を「隠れた状態」でデプロイ → 段階的に有効化

3. 各段階のロールバック手順（問題発生時の戻し方）を定義する
4. 段階展開の承認フロー・確認担当者を定義する

## 出力

implementation-plan.md に追記:

## 段階的デプロイ計画

| フェーズ | 対象 | 確認内容 | 完了条件 | ロールバック手順 |

| Phase 3 | 全社 | 本番確認 | 障害なし 24h | git revert |
