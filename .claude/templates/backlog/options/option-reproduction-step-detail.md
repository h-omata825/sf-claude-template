# option-reproduction-step-detail

## 何をするか

再現手順を 1 操作粒度まで分解して記録する。前提データ・操作ユーザー・タイミングを含めた精密な再現手順が、Sandbox での実証・修正後確認・テストケース設計の品質を左右する。

## 実行手順

1. 課題コメントの再現手順記述を確認する（存在する場合）
2. 再現手順を以下の粒度まで分解して記録する:

   **前提条件**:
   - テストユーザー: プロファイル・権限セット名
   - 前提レコードの状態: オブジェクト名 / ID または条件 / ステータス・フィールド値
   - 前提となる処理の実行状態（例: 「すでに○○ステータスのレコードが存在する」）

   **操作手順**（1 クリック・1 入力が 1 ステップ）:
   1. {アプリ / タブ} を開く
   2. {レコード名・条件} を開く
   3. {ボタン / リンク / 項目名} をクリック / 入力する
   4. ...

   **期待される結果**: {こうなるはず}
   **実際の結果**: {こうなっている・エラーメッセージ等}

3. Sandbox でこの手順を実際に実行して再現を確認する
4. 再現できた場合 → 手順を investigation.md に確定版として記録
5. 再現できない場合 → 条件違いを investigation.md に記録し、差分を不確実点として追加

## 出力

investigation.md「根本原因 / 要件の本質」→「再現条件」セクションを上書き更新:

**前提データ**: {オブジェクト / レコード条件 / 状態}
**操作ユーザー**: {プロファイル}
**操作手順**:
1. ...
2. ...
**期待される結果**: ...
**実際の発生結果**: ...
**Sandbox 再現確認**: 確認済み（{日時}）/ 未確認（{理由}）
