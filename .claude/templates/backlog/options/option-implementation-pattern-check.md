# option-implementation-pattern-check

## 何をするか

実装計画がプロジェクトの既存実装パターン・コーディング規約と整合しているか確認する。新規 LWC / Apex クラス / Flow を作成する時に特に重要。

## 実行手順

1. 実装計画（implementation-plan.md）の主要な実装ポイントを確認する
2. 類似する既存実装を Glob / Grep で特定して Read する:
   - 新規 Apex クラス → 同種の既存 Apex クラス（命名規則・クラス構造・コメントスタイル）
   - 新規 LWC → 同種の既存 LWC（ファイル構成・命名規則・親子コンポーネント構造）
   - 新規 Flow → 同種の既存 Flow（フロー名規則・変数命名・エラーハンドリング方式）
3. 以下の観点で整合性を確認する:

   **命名規則**:
   - API 名・変数名・メソッド名が既存のパターンに沿っているか
   - クラス・ファイルの命名（接頭辞・接尾辞の規則）

   **実装スタイル**:
   - メソッドの責務分割が既存と一致しているか
   - エラーハンドリングの方式が統一されているか
   - コメントの書き方・量

   **共通ユーティリティの活用**:
   - 既存の共通クラス・ユーティリティで代替できる処理を新規実装していないか

4. 乖離がある場合: 意図的な差異か、修正すべき差異かを判断して implementation-plan.md を修正する

## 出力

implementation-plan.md に追記:

## 既存実装パターンとの整合性確認

| 確認観点 | 既存パターン | 今回の計画 | 判定 |
|---|---|---|---|
