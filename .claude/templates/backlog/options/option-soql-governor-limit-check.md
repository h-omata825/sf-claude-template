# option-soql-governor-limit-check

## 何をするか

実装した Apex コードの SOQL ガバナ制限・効率を確認する。SOQL in loop・件数上限超過・インデックス未使用を検出して修正する。

## 実行手順

1. 実装したクラス・トリガーの全 SOQL を抽出する
2. 以下を確認する:

   **SOQL in loop 検出**:
   - for ループ・while ループの中に SOQL が記述されていないか
   - `[SELECT ... FROM ... WHERE Id = :someVar]` がループ内にある → 危険
   - 対策: ループ前に `WHERE Id IN :idSet` で一括取得して Map に変換

   **件数上限確認**:
   - WHERE 条件なしまたは条件が弱い SOQL が全件取得になっていないか
   - 本番 Sandbox で COUNT を確認する（option-soql-dryrun と連携）
   - LIMIT の設定が適切か

   **インデックス**:
   - WHERE 条件に使っているフィールドはインデックス対象か
   - インデックス対象: Id / Name / ExternalId / カスタム一意フィールド / 数式以外
   - 非インデックスフィールドへの条件は大量データで遅延する

   **SOQL 件数上限（1 トランザクション内 100 SOQL）**:
   - 実行フロー全体で SOQL が 100 件を超えないか見積もる

3. 問題を発見したら実装コードを修正する
