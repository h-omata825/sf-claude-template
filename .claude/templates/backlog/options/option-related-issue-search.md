# option-related-issue-search

## 何をするか

Backlog で関連する過去課題の履歴を検索し、同じ機能領域での過去の対応内容・再発パターンを把握する。

## 実行手順

1. 関連キーワードを決める（影響するオブジェクト名・機能名・操作フロー名）
2. `mcp__backlog__get_issues` でキーワード検索する:
   - 同一プロジェクトで過去の完了済み課題を検索
   - 課題種別・ステータスを絞らずに検索する（バグも追加要望も対象）
3. 関連しそうな課題を 3〜5 件 Read して内容を確認する:
   - `mcp__backlog__get_issue` で詳細取得
   - `mcp__backlog__get_issue_comments` でコメント取得
4. 過去課題から以下を抽出する:
   - 同じ機能領域で過去に複数回の修正が入っていないか
   - 過去の修正内容・対応方針
   - 再発のパターン・根本原因への言及
5. 発見した場合は再発防止の観点を investigation.md に追記する

## 出力

investigation.md に追記:

## 関連過去課題

| 課題 ID | 件名（要約） | 対応内容 | 再発との関連性 |
|---|---|---|---|
| ... | ... | ... | 高 / 中 / 低 / なし |

再発パターンが疑われる場合: 根本原因・再発防止の観点を記録する。
