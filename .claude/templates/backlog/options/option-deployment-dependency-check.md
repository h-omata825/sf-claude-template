# option-deployment-dependency-check

## 何をするか

デプロイ順序・依存関係を確認する。オブジェクト → フィールド → Apex → Flow の順など、正しい順序でデプロイしないと失敗するケースを事前に特定する。

## 実行手順

1. 今回の変更で含まれるメタデータ種別を列挙する:
   - カスタムオブジェクト
   - カスタムフィールド
   - Apex クラス・トリガー
   - LWC / Aura
   - Flow / プロセスビルダー
   - 権限セット・プロファイル
   - カスタム設定・Custom Metadata
   - ページレイアウト・コンパクトレイアウト
   - レポート・ダッシュボード

2. 各メタデータ間の依存関係を確認する:
   - Apex クラスが参照するフィールドは先にデプロイが必要
   - Flow が参照する Apex クラスは先にデプロイが必要
   - 権限セットはオブジェクト・フィールドが存在しないと適用できない

3. デプロイ順序を定義する:
   ```
   推奨デプロイ順序:
   1. カスタムオブジェクト / カスタムフィールド（構造定義）
   2. Apex クラス・トリガー（ロジック）
   3. LWC / Aura（UI コンポーネント）
   4. Flow / プロセスビルダー（自動化）
   5. 権限セット / プロファイル（アクセス制御）
   6. ページレイアウト（表示設定）
   ```

4. 一括デプロイで問題ない場合はその旨を記録する
5. 依存関係による分割デプロイが必要な場合はリリース手順書に反映する

## 出力

implementation-plan.md に追記:

## デプロイ依存関係

- 含まれるメタデータ種別: {種別リスト}
- デプロイ順序: 一括可 / 分割要（順序: {1→2→3...}）
- 分割デプロイが必要な理由: {なければ記載不要}
