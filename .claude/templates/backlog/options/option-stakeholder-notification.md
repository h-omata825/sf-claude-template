# option-stakeholder-notification

## 何をするか

関係者への完了通知を実施する。Backlog コメント・Slack・メール等での周知を行う。

## 実行手順

1. 通知先を確認する:
   - 課題の依頼者（Backlog 課題作成者）
   - コメントで要件を追加した関係者
   - 影響を受けるユーザーの担当者・マネージャー
   - 他チームの担当者（横断影響がある場合）

2. 通知内容を準備する（業務ユーザー向け・技術的な詳細は不要）:
   ```
   件名: 【対応完了】{課題件名}

   お疲れ様です。

   ご依頼いただきました「{課題件名の要約}」の対応が完了しました。

   ■ 対応内容
   {変更の概要を業務的に説明}

   ■ 有効日時
   {リリース日時}

   ■ 確認方法
   {どの画面・操作で確認できるか}

   ご不明点があればお知らせください。
   ```

3. 通知方法に応じて実施する:
   - **Backlog コメント**: `mcp__backlog__add_issue_comment` でコメントを追加する
   - **Slack / メール**: communicator エージェントに文章作成を依頼する（テキスト出力で案内）

4. 通知実施済みを test-report.md に記録する

## 出力

test-report.md に追記:

## ステークホルダー通知

- 通知先: {氏名 / グループ名}
- 通知方法: Backlog コメント / Slack / メール
- 通知日時: {YYYY-MM-DD HH:MM}
- 対応: 通知済み
