# option-acceptance-criteria-recheck

## 何をするか

課題コメントの追加要件・ステークホルダーの期待との整合を最終確認する。「これも対応して」という後付けコメントを漏らさないようにする。

## 実行手順

1. 課題の全コメントを `mcp__backlog__get_issue_comments` で取得して読み返す
2. 以下を確認する:

   **後付け要件の確認**:
   - コメントに「〇〇も対応してください」「〇〇に変更してください」がないか
   - 課題作成後に仕様が変更されたコメントがないか
   - 「確認後に依頼」「後で追加」等の言及がないか

   **受入基準の整合**:
   - 課題本文の「期待する結果」「完了条件」が全て達成されているか
   - 依頼者（課題作成者・コメント者）の期待に応えているか

   **スコープの確認**:
   - コメントで「対象外にしてください」「今回は〇〇だけでよい」とスコープ縮小された内容がないか
   - 縮小されたスコープを正しく守っているか

3. 漏れを発見した場合: 対応内容を implementation-plan.md に追記して実装する
4. 今回対応しない項目がある場合: 別課題として Backlog に登録することを提案する

## 出力

test-report.md に追記:

## 受入基準再確認

- コメント件数確認: {N} 件を全件確認
- 後付け要件: なし / あり（{内容・対応済み / 別課題化}）
- 受入基準充足: 全項目 OK / 未対応あり（{内容・対応済み}）
- 最終判定: リリース可 / 追加実装要
