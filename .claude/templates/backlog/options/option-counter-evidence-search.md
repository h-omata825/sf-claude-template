# option-counter-evidence-search

## 何をするか

現在の原因仮説に矛盾する反証を積極的に探す。確証バイアス（仮説を支持する証拠だけ集める傾向）を防ぐ。

## 実行手順

1. 現在の採用原因仮説を 1 文で明示する（例: 「FLS 設定が原因でフィールドが表示されない」）
2. この仮説が**間違いだとしたら**、どんな証拠が見つかるはずかを考える:
   - 「FLS が原因なら、管理者でも同じ症状が出るはず」→ 管理者では症状が出ないなら FLS が原因
   - 「コードのバグが原因なら、特定条件でのみ発生するはず」→ 全ユーザーで発生するなら別の原因
3. 反証を探す具体的なアクションを実行する:
   - 別ユーザー・別プロファイルでの動作確認（Sandbox）
   - 別オブジェクト・別レコードでの動作確認
   - 条件を変えた場合の動作確認
   - コードの他の分岐・フローでの確認
4. 反証が見つかった場合:
   - 仮説を修正または破棄する
   - 新しい仮説を立てて同じプロセスを繰り返す
5. 反証が見つからなかった場合:
   - 仮説の信頼度を上げて採用する

## 出力

investigation.md「原因仮説」セクションに追記:

## 反証探索の結果

- 探索した反証: {どんな証拠を探したか}
- 結果: 反証なし（仮説 A の信頼度: 高）/ 反証あり（{内容}→仮説修正: {修正後仮説}）
