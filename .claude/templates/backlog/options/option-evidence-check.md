# option-evidence-check

## 何をするか

実装前 Before エビデンス（画面キャプチャ・データ状態）の取得状況を確認する。After エビデンスと比較するための Before が揃っているかを確認する。

## 実行手順

1. 課題対応に必要な Before エビデンスのリストを確認する:
   - 症状が発生している画面のキャプチャ（エラーメッセージ・表示状態）
   - 問題のあるデータの状態（レコードの値・件数）
   - 変更前の設定状態（管理画面の設定値）

2. investigation.md・エビデンスファイルを確認して、以下が揃っているかチェックする:
   - [ ] 症状の画面キャプチャ（またはエラーログ）
   - [ ] 変更前のデータ状態（必要な場合）
   - [ ] 変更前の設定状態（管理画面設定変更がある場合）

3. 不足しているエビデンスがある場合:
   - Sandbox で確認できるものは Sandbox でキャプチャを取得する
   - 本番データが必要な場合はユーザーに依頼する

4. 取得状況を validation-report.md に記録する

## 出力

validation-report.md に追記:

Before エビデンス充足状況: OK → 実装進行 / 不足（{追加取得が必要なもの}）
