# option-feature-flag-design

## 何をするか

カスタム設定または Custom Metadata で機能 ON/OFF を制御するフィーチャーフラグを設計する。段階的展開・一部ユーザー先行展開を実現する。

## 実行手順

1. フィーチャーフラグで制御する機能の範囲を定義する
2. 制御方式を選択する:

   **Custom Metadata（推奨）**:
   - デプロイ可能・バージョン管理可
   - 環境別に値を変えられる
   - 組織全体 or レコード単位での ON/OFF

   **カスタム設定**:
   - 実行時に変更可能（コードデプロイ不要）
   - 階層型（組織・プロファイル・ユーザー別）の ON/OFF が可能

3. フラグのデータ設計を定義する:
   - オブジェクト名・フィールド名（API 名）
   - デフォルト値（ON / OFF どちらを安全側にするか）
   - 制御粒度（全社 / プロファイル別 / ユーザー別）

4. Apex での参照コードを設計する:
   ```apex
   // 例: Custom Metadata 参照
   Feature_Flag__mdt flag = Feature_Flag__mdt.getInstance('新機能名');
   if (flag != null && flag.Is_Enabled__c) {
       // 新機能の処理
   }
   ```

5. フラグ有効化・無効化の手順書を作成する（誰が・いつ・どう操作するか）

## 出力
