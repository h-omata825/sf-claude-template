# option-permission-test

## 何をするか

異なるプロファイル・権限セットでの動作確認を実施する。特定のユーザーだけ動作しない・見えないというケースを発見する。

## 実行手順

1. 影響するプロファイル・権限セットを確認する（option-permission-fls-check の結果）
2. 以下のユーザーパターンでテストする:
   - システム管理者: 全機能が動作する（ベースライン）
   - 担当ユーザー（主なプロファイル）: 業務フローが動作する
   - 閲覧のみユーザー（ReadOnly プロファイル等）: 編集できないことを確認
   - 権限なしユーザー: アクセスできないことを確認
3. 各ユーザーで以下を確認する:
   - フィールドが表示される / されない
   - ボタン・アクションが表示される / されない
   - データの読み取り・更新ができる / できない
   - 期待するエラーメッセージが表示される（権限なし）
4. 問題を発見した場合:
   - FLS 設定の変更 → 権限セット・プロファイルの FLS を修正する
   - Apex の with sharing の問題 → 実装を修正する

## 出力

test-report.md に追記:
