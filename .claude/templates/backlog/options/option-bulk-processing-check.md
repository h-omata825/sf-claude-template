# option-bulk-processing-check

## 何をするか

Apex トリガー・バッチで 200 件超の処理耐性を確認する。バルク処理対応の実装になっているか検証する。

## 実行手順

1. 実装したトリガー・Apex クラスのバルク処理対応を確認する:

   **トリガーのバルク対応**:
   - `Trigger.new` / `Trigger.old` はリスト（複数件）として扱っているか
   - ループ内で 1 件ずつ処理していないか（単一処理は 200 件で限界超過）
   - 対策: Set / List / Map でデータを事前収集してから一括処理

   **DML のバルク化**:
   - 1 件ずつ INSERT / UPDATE していないか
   - 対策: リストに追加してからループ外で一括 DML

   **SOQL のバルク化**:
   - ループ内で単一 ID を検索していないか
   - 対策: ID を Set に収集してから `WHERE Id IN :idSet` で一括取得

2. バルク処理確認テストを実行する（Sandbox で 200 件以上のデータを使ったテスト）:
   ```bash
   # テストクラスで 200 件以上のデータを作成するメソッドを確認
   Grep pattern: "for\s*\(Integer i = 0"
   ファイル: force-app/**/*Test*.cls
   ```

3. バルク処理問題を発見した場合はコードを修正する
