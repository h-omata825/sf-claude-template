# option-code-comments-detail

## 何をするか

複雑なロジック・前提条件・設計判断の意図をコードコメントとして残す。後続の開発者が「なぜこう書いてあるか」を理解できるようにする。

## 実行手順

1. 実装したコードを読み返して、以下の箇所を特定する:
   - 複雑な条件分岐（分岐数 3 以上）
   - パフォーマンス最適化のためのトリッキーな実装
   - Salesforce の制約・制限を回避するための回避策
   - 業務ルールが直接コードに埋め込まれている箇所
   - 「将来変更可能性が高い」と判断した箇所

2. 以下のルールでコメントを追加する:
   - **何をしているか**: コードを読めば分かる場合は書かない
   - **なぜこうしているか**: 書くべき内容（制約・業務ルール・トレードオフ）
   - **前提条件**: このメソッドが正しく動くための前提
   - **注意点**: 変更する際に注意すべきこと

3. コメントの例:
   ```apex
   // ガバナ制限対策: ループ前に一括取得して Map に変換
   // SOQL in loop になるとトリガーで 200 件超の場合に上限超過する
   Map<Id, Account> accountMap = new Map<Id, Account>(
       [SELECT Id, Name FROM Account WHERE Id IN :accountIds]
   );
   ```

4. 過剰なコメント（コードを読めば明白なこと）は書かない

## 出力

特になし（コードに直接追加するため）。コメント追加後、レビュー観点で再読して過不足を確認する。
