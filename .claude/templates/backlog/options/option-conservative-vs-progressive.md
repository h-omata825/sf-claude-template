# option-conservative-vs-progressive

## 何をするか

保守的案（場当たり修正）と抜本案（根本対応）を比較して、今回の修正レベルを選択する。再発リスクが高い場合に重要。

## 実行手順

1. option-similar-past-issue または option-causal-chain-analysis の結果を確認する（再発パターン・根本原因）
2. 以下の 2 案を整理する:

   **保守的案（場当たり修正）**:
   - 今回の症状の直接原因だけを修正する
   - 変更範囲: 最小限
   - リスク: 同じ根本原因から別の症状が発生する可能性が残る
   - 工数: 小

   **抜本案（根本対応）**:
   - 根本原因まで修正する（設計・構造・データを含む）
   - 変更範囲: 広い
   - リスク: 影響範囲が広いため別の問題を引き起こす可能性
   - 工数: 大

3. 以下の判断軸で適切な修正レベルを決定する:
   - 再発リスクの高さ（過去に同種の問題が繰り返されているか）
   - 根本対応の影響範囲（どれだけのテスト・確認が必要か）
   - 今回の優先度・期限
   - チームのリスク許容度

4. 決定した修正レベルと根拠を approach-plan.md に記録する
