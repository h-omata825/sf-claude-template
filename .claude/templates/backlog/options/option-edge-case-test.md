# option-edge-case-test

## 何をするか

境界値・異常系テストを実施する。null / 空文字 / 最大件数 / 権限なしユーザーなど、エッジケースでの挙動を確認する。

## 実行手順

1. 変更内容のエッジケースを列挙する:
   - **入力値の境界**:
     - null 値・空文字を渡した場合
     - 最小値・最大値（数値・日付・文字列長）
     - 許可されていない文字・フォーマット
   - **データ状態の境界**:
     - レコードが存在しない場合（クエリ結果 0 件）
     - 重複レコードがある場合
     - 関連レコードが欠損している場合（FK が null）
   - **権限の境界**:
     - 権限を持たないユーザーが操作した場合
     - FLS で非表示のフィールドを持つユーザー
   - **量の境界**:
     - 0 件・1 件・200 件・大量件数での動作

2. 各エッジケースを Sandbox で確認する（テストクラスまたは手動）
3. 問題が発見された場合はコードを修正する
