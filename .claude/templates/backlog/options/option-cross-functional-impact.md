# option-cross-functional-impact

## 何をするか

リリース直前に横断機能への影響を最終確認する。テスト段階の集中で見落とされがちな、他機能・他チームへの影響を最後に確認する。

## 実行手順

1. 変更内容のサマリーを再確認する（何を変えたか）
2. 以下の観点で横断的な影響を確認する:

   **共通コンポーネントへの影響**:
   - 変更した Apex クラス・LWC が別機能からも使われていないか（Grep で確認）
   - 共通ユーティリティを変更した場合: 利用箇所を全て確認した上で問題なしと言えるか

   **データ整合性**:
   - 変更後のデータ状態が他の機能の処理に影響しないか
   - 新規フィールドの値が NULL / デフォルト値の場合、他の機能で問題が起きないか

   **他チーム・他プロジェクトへの影響**:
   - 複数チームが同じ組織で開発している場合: 他チームが変更対象を使っていないか
   - 進行中の別プロジェクトと変更が競合しないか

   **UI の一貫性**:
   - 変更した画面デザイン・操作フローが全体の UX と一貫しているか

3. 問題を発見した場合: リリース前に対応する

## 出力

test-report.md に追記:

## 横断機能影響確認

- 確認対象: {確認した機能・コンポーネント}
- 問題: なし / あり（{詳細・対応済み内容}）
- 最終判定: リリース可 / 追加対応要（{内容}）
