# option-similar-past-issue

## 何をするか

同じ機能領域で同種の課題が過去に繰り返されていないか確認する。再発パターンの発見が目的。

## 実行手順

1. 現課題の機能領域・症状のキーワードを 3〜5 個抽出する
2. Backlog で過去完了課題を検索する:
   - `mcp__backlog__get_issues` でキーワード検索（完了済み含む）
   - 特に同じオブジェクト・機能に関連する課題を探す
3. 過去課題を読んで以下を確認する:
   - 今回と同じ根本原因の可能性があるか
   - 過去の対応で「場当たり修正」になっており根本対応していないか
   - 同じ機能で 2 回以上の修正履歴があるか
4. git log でも同一ファイルへの繰り返し修正を確認する:
   ```bash
   git log --oneline -20 -- {変更対象ファイルパス}
   ```
5. 再発パターンを発見した場合:
   - 根本対応の必要性を investigation.md に明記
   - option-conservative-vs-progressive（保守的案 vs 抜本案比較）の実行を推奨

## 出力

investigation.md に追記:

## 類似過去課題の再発確認

- 同機能領域での過去修正: {件数} 件
- 再発パターン: あり / なし
- 根本対応の必要性: 高 / 低（{理由}）
