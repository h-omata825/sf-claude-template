# option-similar-impl-search

## 何をするか

変更対象と同じパターンの処理を `force-app/` から全て探して読む。実装判断の精度を最も左右するステップ。

## 実行手順

1. 変更内容から「類似パターン」を言語化する:
   - 「下書き保存機能追加」→ 別モーダル・別オブジェクトの下書き保存処理
   - 「プリチェックロジック修正」→ 他ビザ種別の同種チェック処理
   - 「Chatter メンション処理修正」→ 他オブジェクトの同種メンション処理
2. Glob / Grep で類似ファイルを特定する
3. 特定したファイルを全読みして、実装パターン（処理方式・命名・構造）を把握する
4. 類似実装と今回の変更計画の整合性を確認する:
   - 整合している → 同パターンで実装する旨を記録
   - 意図的に差異がある → 差異の理由を明記
   - 不整合 → 業務要件の不確実点として記録

## 出力

investigation.md「類似する既存実装」セクションに追記:

| 実装名 | ファイルパス | このケースとの関連 | 実装パターン |
|---|---|---|---|
| ... | ... | ... | ... |

類似実装が見つからない場合は「なし（探索済み）」と明記する。
