# option-data-migration-plan

## 何をするか

既存レコードの一括更新・新項目バックフィル等のデータ移行計画を策定する。コード変更だけでは対応できないデータ側の整合性を確保する。

## 実行手順

1. データ移行が必要な内容を特定する:
   - 新規必須フィールド追加（既存レコードへのデフォルト値設定）
   - フィールド統廃合（旧フィールドから新フィールドへのデータ移行）
   - ステータス値変更（既存レコードの一括更新）
   - オブジェクト構造変更（データの再配置）

2. 移行対象データを調査する:
   - 対象レコード数（Sandbox で COUNT クエリを実行）
   - 移行元・移行先のフィールド定義
   - NULL / 空値 の扱い方

3. 移行手法を選択する:
   - **Data Loader**: 〜10 万件、手動実行
   - **Batch Apex**: 10 万件超、自動化・再実行可
   - **Flow（一括起動）**: 件数が少ない場合、ノーコード

4. 移行計画を詳細化する:
   - 移行前バックアップ手順（Data Export / Data Loader で CSV 出力）
   - 移行実行タイミング（本番リリース前後）
   - バリデーション（移行後に件数・値を確認するクエリ）
   - ロールバック手順（バックアップから戻す手順）

5. 移行スクリプト / Batch クラスの設計（必要な場合）

## 出力

implementation-plan.md に追記:

## データ移行計画

- 対象オブジェクト: {ObjectName__c}（Sandbox 件数: {N} 件）
- 移行内容: {旧フィールド} → {新フィールド}（{変換ロジック}）
- 移行手法: Data Loader / Batch Apex / Flow
- 実行タイミング: デプロイ直後 / デプロイ前
- 確認クエリ: `SELECT COUNT() FROM {Object} WHERE {移行完了条件}`
- ロールバック: {バックアップ CSV からの Data Loader インポート}
