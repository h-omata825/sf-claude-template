# レポート / ダッシュボード（Reports / Dashboards）

## レポート作成時

1. `docs/data/reports-dashboards.md` で類似レポートが既にないか確認
2. レポートタイプ選定: 集計が必要か（サマリー / マトリックス）、詳細一覧か（表形式）、結合が必要かを確認

**確認事項:**
- フィルター条件: 日付範囲・担当者・レコードタイプ等
- グルーピング: サマリー/マトリックスの場合、何で集計するか
- 保存先フォルダ: 公開 / 個人 / チーム
- ダッシュボードに追加するか

**注意:** クロスフィルター・バケット項目・カスタムサマリー数式は複雑になりがちなので要件を再確認

## ダッシュボード作成時

- ダッシュボードの閲覧者（実行ユーザー）を確認: 「閲覧者のデータ」vs「固定ユーザーのデータ」
- 自動更新頻度の確認（毎日/毎週/手動）
