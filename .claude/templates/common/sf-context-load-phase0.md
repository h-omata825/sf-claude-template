# sf-context-loader 呼び出し仕様（Phase 0 共通）

## 概要

`sf-context-loader` は `docs/` 配下のファイルを横断検索し、タスクに関連する業務コンテキスト（オブジェクト定義・UC仕様・設計意図・注意点）の要約を返す共通エージェント。

## 呼び出しパラメータ

| パラメータ | 説明 | 必須 |
|---|---|---|
| `task_description` | ユーザー指示またはタスクの概要（「何をするか」を日本語で記述） | 必須 |
| `project_dir` | プロジェクトルートパス（`docs/` の親ディレクトリ）。不明な場合はカレントディレクトリを指定 | 必須 |
| `focus_hints` | 対象を絞り込むキーワード（オブジェクト名・機能名等）の配列。空配列 `[]` で docs/ 全体を検索 | 任意 |

## 結果の標準解釈

- **「該当コンテキストなし」が返った場合**: このステップをスキップして次の手順へ（`docs/` 未整備または SF 無関係タスク）
- **関連コンテキストが返った場合**: 返された内容（関連オブジェクト・UC・ビジネスルール・注意点）を以降の判断材料として保持する

各 agent 固有の `task_description` ヒント・スキップ先・コンテキスト利用方法は、各 agent の Phase 0 ブロックに記載する。
